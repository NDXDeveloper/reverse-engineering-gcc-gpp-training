# 25.4 — Writing a Standalone Python Parser/Serializer

> 🎯 **Goal of this section**: implement a Python parser capable of reading any CFR archive, and a serializer capable of producing new ones. The ultimate test is the **round-trip**: a file generated by our Python code must be accepted and correctly read by the original binary.

---

## Implementation Strategy

We could write the parser in one go, but it is safer to proceed layer by layer, validating each layer independently before moving on to the next:

1. **CRC layer** — implement the two CRC variants used by the format (standard CRC-32 and CRC-16/CCITT init=0x1D0F). Test them in isolation on known data.  
2. **XOR layer** — implement the rotating XOR transformation. Test it by decrypting a known block.  
3. **Parsing layer** — read the header, records, and footer. Validate CRCs at each step.  
4. **Serialization layer** — write a complete CFR file from in-memory data.  
5. **Round-trip test** — read an archive, rewrite it, verify that the original binary accepts it.

This progression ensures that each building block is solid before being integrated into the whole. A bug in the CRC-16 that would only be detected during the final round-trip would be much harder to diagnose.

---

## The Foundations: CRC and XOR

### CRC-32

The format uses a classic CRC-32 — polynomial `0xEDB88320` (reflected form), initial value `0xFFFFFFFF`, final XOR `0xFFFFFFFF`. This is exactly the variant implemented by Python's `binascii` module:

```python
import binascii

def crc32(data: bytes) -> int:
    """Standard CRC-32 (ISO 3309), identical to zlib/binascii."""
    return binascii.crc32(data) & 0xFFFFFFFF
```

The `& 0xFFFFFFFF` ensures an unsigned 32-bit result, regardless of the Python version.

Let's verify immediately on the header of `demo.cfr`:

```python
with open("samples/demo.cfr", "rb") as f:
    header_bytes = f.read(32)

# The header CRC is stored at offset 0x10 (4 bytes, little-endian)
import struct  
stored_crc = struct.unpack_from("<I", header_bytes, 0x10)[0]  

# The CRC is computed over the first 16 bytes, with the CRC field set to zero
check_data = bytearray(header_bytes[:16])
# Bytes 0x10–0x13 are NOT part of the first 16 bytes,
# so no need to zero them out — they are not included.
computed_crc = crc32(bytes(check_data))

assert stored_crc == computed_crc, \
    f"Header CRC mismatch: stored={stored_crc:#010x} computed={computed_crc:#010x}"
print(f"Header CRC OK: {computed_crc:#010x}")
```

If this assertion passes, our CRC-32 implementation is validated on the real case. If it fails, we need to go back and verify the exact range of bytes covered by the CRC (this is the classic pitfall: does the CRC cover the first 16 bytes, or the first 16 bytes with the CRC field zeroed out?).

> 💡 **Common pitfall**: some implementations compute the header CRC by including the `header_crc` field itself, set to zero. Others purely exclude this field and only compute over the bytes that precede it. The difference is subtle but produces different results. In section 25.2, we formulated the hypothesis that the CRC covers the first 16 bytes (offsets `0x00`–`0x0F`), i.e., exactly the fields `magic + version + flags + num_records + timestamp`. The `header_crc` field starts at offset `0x10` and is therefore not included in the computation.

### CRC-16/CCITT (init=0x1D0F)

This variant does not exist in the Python standard library. We implement it manually:

```python
def crc16_ccitt(data: bytes, init: int = 0x1D0F) -> int:
    """CRC-16/CCITT with polynomial 0x1021 and custom initial value."""
    crc = init
    for byte in data:
        crc ^= byte << 8
        for _ in range(8):
            if crc & 0x8000:
                crc = (crc << 1) ^ 0x1021
            else:
                crc = crc << 1
            crc &= 0xFFFF
    return crc
```

The initial value `0x1D0F` is the peculiarity of this format. The standard CRC-16/CCITT variant uses `0xFFFF`; here, the format's author chose a non-standard value. This is exactly the kind of detail that can only be discovered through reverse engineering — no public documentation mentions it since the format is proprietary.

Let's validate on a known record from `packed_noxor.cfr` (no XOR, so the data is in cleartext and the CRC is computed directly over it):

```python
# Read the first record from packed_noxor.cfr
with open("samples/packed_noxor.cfr", "rb") as f:
    f.seek(32)  # skip the header
    rec_type, rec_flags, name_len, data_len = struct.unpack("<BBHI", f.read(8))
    name = f.read(name_len)
    data = f.read(data_len)
    stored_crc16 = struct.unpack("<H", f.read(2))[0]

# CRC-16 computed over name + data (original data, before any XOR)
computed_crc16 = crc16_ccitt(name + data)

assert stored_crc16 == computed_crc16, \
    f"Record CRC-16 mismatch: stored={stored_crc16:#06x} computed={computed_crc16:#06x}"
print(f"Record CRC-16 OK: {computed_crc16:#06x}")
```

### XOR Transformation

The transformation is a rotating XOR with a 4-byte key:

```python
XOR_KEY = bytes([0x5A, 0x3C, 0x96, 0xF1])

def xor_transform(data: bytes) -> bytes:
    """Applies/removes the rotating XOR. The operation is its own inverse."""
    key_len = len(XOR_KEY)
    return bytes(b ^ XOR_KEY[i % key_len] for i, b in enumerate(data))
```

The elegance of XOR is that the same function serves to encrypt and decrypt. Let's verify:

```python
original = b"Hello from the CFR archive format!"  
encrypted = xor_transform(original)  
decrypted = xor_transform(encrypted)  
assert decrypted == original  
print(f"XOR round-trip OK")  
```

## The Parser: Reading a CFR Archive

### Data Structures

Before parsing, let's define classes to represent the contents in memory. We use `dataclass` for readability:

```python
import struct  
from dataclasses import dataclass, field  
from typing import List, Optional  
from enum import IntEnum, IntFlag  

class RecordType(IntEnum):
    TEXT   = 0x01
    BINARY = 0x02
    META   = 0x03

class HeaderFlags(IntFlag):
    XOR_ENABLED = 1 << 0
    HAS_FOOTER  = 1 << 1

@dataclass
class CFRHeader:
    magic: bytes            # 4 bytes, b"CFRM"
    version: int            # uint16
    flags: HeaderFlags      # uint16
    num_records: int        # uint32
    timestamp: int          # uint32
    header_crc: int         # uint32
    author: str             # 8 bytes, null-padded
    data_len_xor: int       # uint32 (XOR of all data_len values)

@dataclass
class CFRRecord:
    rec_type: int           # uint8 (RecordType)
    rec_flags: int          # uint8
    name: str               # variable length
    data: bytes             # variable length, AFTER de-XOR
    stored_crc16: int       # uint16

@dataclass
class CFRFooter:
    magic: bytes            # 4 bytes, b"CRFE"
    total_size: int         # uint32
    global_crc: int         # uint32

@dataclass
class CFRArchive:
    header: CFRHeader
    records: List[CFRRecord] = field(default_factory=list)
    footer: Optional[CFRFooter] = None
```

### Parsing Function

```python
import io

HEADER_SIZE = 32  
FOOTER_SIZE = 12  
REC_HEADER_SIZE = 8  
MAX_RECORDS = 1024  

class CFRParseError(Exception):
    """Error while parsing a CFR archive."""
    pass

def parse_cfr(filepath: str) -> CFRArchive:
    """Parses a CFR archive and returns its in-memory representation."""
    with open(filepath, "rb") as f:
        raw = f.read()

    if len(raw) < HEADER_SIZE:
        raise CFRParseError(f"File too short for a header ({len(raw)} bytes)")

    buf = io.BytesIO(raw)

    # ── Header ──────────────────────────────────────────────
    hdr_bytes = buf.read(HEADER_SIZE)
    # Let's manually extract each field for clarity
    # (a single struct.unpack on 32 bytes would be possible but fragile):
    magic       = hdr_bytes[0:4]
    version     = struct.unpack_from("<H", hdr_bytes, 4)[0]
    flags       = HeaderFlags(struct.unpack_from("<H", hdr_bytes, 6)[0])
    num_records = struct.unpack_from("<I", hdr_bytes, 8)[0]
    timestamp   = struct.unpack_from("<I", hdr_bytes, 12)[0]
    header_crc  = struct.unpack_from("<I", hdr_bytes, 16)[0]
    author_raw  = hdr_bytes[20:28]
    dlx_raw     = hdr_bytes[28:32]

    if magic != b"CFRM":
        raise CFRParseError(f"Invalid magic: {magic!r} (expected: b'CFRM')")

    if num_records > MAX_RECORDS:
        raise CFRParseError(f"Too many records: {num_records} (max {MAX_RECORDS})")

    # Header CRC verification (over the first 16 bytes)
    expected_crc = crc32(hdr_bytes[:16])
    if header_crc != expected_crc:
        raise CFRParseError(
            f"Invalid header CRC: stored={header_crc:#010x} "
            f"computed={expected_crc:#010x}"
        )

    author = author_raw.rstrip(b"\x00").decode("ascii", errors="replace")
    data_len_xor = struct.unpack_from("<I", dlx_raw, 0)[0]

    header = CFRHeader(
        magic=magic, version=version, flags=flags,
        num_records=num_records, timestamp=timestamp,
        header_crc=header_crc, author=author,
        data_len_xor=data_len_xor
    )

    do_xor = bool(flags & HeaderFlags.XOR_ENABLED)

    # ── Records ─────────────────────────────────────────────
    records = []
    dlx_check = 0

    for i in range(num_records):
        rh_bytes = buf.read(REC_HEADER_SIZE)
        if len(rh_bytes) < REC_HEADER_SIZE:
            raise CFRParseError(f"Record {i}: truncated header")

        rec_type  = rh_bytes[0]
        rec_flags = rh_bytes[1]
        name_len  = struct.unpack_from("<H", rh_bytes, 2)[0]
        data_len  = struct.unpack_from("<I", rh_bytes, 4)[0]

        name_bytes = buf.read(name_len)
        if len(name_bytes) < name_len:
            raise CFRParseError(f"Record {i}: truncated name")

        raw_data = buf.read(data_len)
        if len(raw_data) < data_len:
            raise CFRParseError(f"Record {i}: truncated data")

        crc_bytes = buf.read(2)
        if len(crc_bytes) < 2:
            raise CFRParseError(f"Record {i}: truncated CRC-16")
        stored_crc16 = struct.unpack("<H", crc_bytes)[0]

        # De-XOR data if necessary
        if do_xor and data_len > 0:
            plain_data = xor_transform(raw_data)
        else:
            plain_data = raw_data

        # CRC-16 verification (computed over name + ORIGINAL data)
        expected_crc16 = crc16_ccitt(name_bytes + plain_data)
        if stored_crc16 != expected_crc16:
            raise CFRParseError(
                f"Record {i} ({name_bytes!r}): invalid CRC-16: "
                f"stored={stored_crc16:#06x} computed={expected_crc16:#06x}"
            )

        name = name_bytes.decode("ascii", errors="replace")
        records.append(CFRRecord(
            rec_type=rec_type, rec_flags=rec_flags,
            name=name, data=plain_data,
            stored_crc16=stored_crc16
        ))

        dlx_check ^= data_len

    # Verify the data_len_xor field
    if dlx_check != data_len_xor:
        raise CFRParseError(
            f"Invalid data_len_xor: stored={data_len_xor:#010x} "
            f"computed={dlx_check:#010x}"
        )

    # ── Footer ──────────────────────────────────────────────
    footer = None
    if flags & HeaderFlags.HAS_FOOTER:
        ftr_bytes = buf.read(FOOTER_SIZE)
        if len(ftr_bytes) == FOOTER_SIZE:
            ftr_magic = ftr_bytes[0:4]
            ftr_total = struct.unpack_from("<I", ftr_bytes, 4)[0]
            ftr_crc   = struct.unpack_from("<I", ftr_bytes, 8)[0]

            if ftr_magic != b"CRFE":
                raise CFRParseError(
                    f"Invalid footer magic: {ftr_magic!r} (expected: b'CRFE')"
                )

            if ftr_total != len(raw):
                raise CFRParseError(
                    f"Inconsistent footer total_size: "
                    f"stored={ftr_total} actual size={len(raw)}"
                )

            # Global CRC = CRC-32 of everything preceding the footer
            payload = raw[:len(raw) - FOOTER_SIZE]
            expected_global = crc32(payload)
            if ftr_crc != expected_global:
                raise CFRParseError(
                    f"Invalid footer global CRC: "
                    f"stored={ftr_crc:#010x} computed={expected_global:#010x}"
                )

            footer = CFRFooter(
                magic=ftr_magic,
                total_size=ftr_total,
                global_crc=ftr_crc
            )

    return CFRArchive(header=header, records=records, footer=footer)
```

### Design Points to Note

**The order of XOR/CRC operations is crucial.** The CRC-16 is verified on data *after* de-XOR (i.e., on the original data). If we reversed the order — computing the CRC on data still XOR-ed — the verification would systematically fail on XOR archives. This is information obtained in section 25.2 by comparing CRCs between XOR and non-XOR archives.

**Parsing is sequential, not offset-based.** We read the header, then the records one by one following the cursor. Each record's `name_len` and `data_len` fields determine how many bytes to advance. There is no offset table in the format — it's a linear design. The implication is that a file corrupted in the middle renders the rest unreadable.

**Errors are explicit exceptions.** For reverse engineering, it is preferable to crash loudly on each anomaly rather than attempting silent recovery. Each `CFRParseError` points to a specific format invariant.

## The Serializer: Writing a CFR Archive

### Serialization Function

```python
import time

def serialize_cfr(
    records: List[CFRRecord],
    author: str = "python",
    xor_enabled: bool = True,
    include_footer: bool = True,
    timestamp: Optional[int] = None,
) -> bytes:
    """Serializes a list of records into a complete CFR archive."""

    if len(records) > MAX_RECORDS:
        raise ValueError(f"Too many records: {len(records)} (max {MAX_RECORDS})")

    flags = HeaderFlags(0)
    if xor_enabled:
        flags |= HeaderFlags.XOR_ENABLED
    if include_footer:
        flags |= HeaderFlags.HAS_FOOTER

    ts = timestamp if timestamp is not None else int(time.time())

    # ── Serialize the records into a buffer ─────────────────
    records_buf = io.BytesIO()
    dlx = 0

    for rec in records:
        name_bytes = rec.name.encode("ascii")
        name_len = len(name_bytes)
        data_len = len(rec.data)
        dlx ^= data_len

        # Record header
        rh = struct.pack("<BBHI", rec.rec_type, rec.rec_flags, name_len, data_len)
        records_buf.write(rh)

        # Name (never transformed)
        records_buf.write(name_bytes)

        # CRC-16 over name + original data (BEFORE XOR)
        rec_crc = crc16_ccitt(name_bytes + rec.data)

        # Data (XOR if flag is active)
        if xor_enabled and data_len > 0:
            records_buf.write(xor_transform(rec.data))
        else:
            records_buf.write(rec.data)

        # CRC-16
        records_buf.write(struct.pack("<H", rec_crc))

    records_bytes = records_buf.getvalue()

    # ── Build the header ────────────────────────────────────
    #
    # We need to compute header_crc over the first 16 bytes.
    # Since header_crc occupies bytes 16–19 (after the first 16),
    # it does not include itself.

    author_padded = author.encode("ascii")[:8].ljust(8, b"\x00")
    dlx_bytes = struct.pack("<I", dlx)

    # First 16 bytes (magic + version + flags + num_records + timestamp)
    hdr_prefix = struct.pack("<4sHHII",
        b"CFRM", 2, int(flags), len(records), ts
    )
    assert len(hdr_prefix) == 16

    hdr_crc = crc32(hdr_prefix)

    header_bytes = hdr_prefix + struct.pack("<I", hdr_crc) + author_padded + dlx_bytes
    assert len(header_bytes) == HEADER_SIZE

    # ── Assemble without footer ─────────────────────────────
    payload = header_bytes + records_bytes

    # ── Footer (optional) ───────────────────────────────────
    if include_footer:
        total_size = len(payload) + FOOTER_SIZE
        global_crc = crc32(payload)
        footer_bytes = struct.pack("<4sII", b"CRFE", total_size, global_crc)
        return payload + footer_bytes
    else:
        return payload
```

### Helpers for Creating Records

To simplify record construction, let's add a few utility functions:

```python
def make_text_record(name: str, text: str) -> CFRRecord:
    """Creates a TEXT type record."""
    return CFRRecord(
        rec_type=RecordType.TEXT, rec_flags=0,
        name=name, data=text.encode("utf-8"),
        stored_crc16=0  # will be recomputed during serialization
    )

def make_binary_record(name: str, data: bytes) -> CFRRecord:
    """Creates a BINARY type record."""
    return CFRRecord(
        rec_type=RecordType.BINARY, rec_flags=0,
        name=name, data=data,
        stored_crc16=0
    )

def make_meta_record(name: str, metadata: dict) -> CFRRecord:
    """Creates a META type record from a key=value dictionary."""
    text = "\n".join(f"{k}={v}" for k, v in metadata.items())
    return CFRRecord(
        rec_type=RecordType.META, rec_flags=0,
        name=name, data=text.encode("utf-8"),
        stored_crc16=0
    )
```

## The Round-Trip Test

The round-trip is the definitive validation: we prove that our implementation conforms to the format by verifying that the original binary accepts the files we produce.

### Round-trip #1: Read then Rewrite

```python
def test_roundtrip_read_write(filepath: str, output_path: str):
    """Reads an archive, rewrites it, verifies that the result is identical."""
    # 1. Read the original archive
    archive = parse_cfr(filepath)

    # 2. Rewrite with the same parameters
    xor = bool(archive.header.flags & HeaderFlags.XOR_ENABLED)
    footer = archive.footer is not None
    output = serialize_cfr(
        archive.records,
        author=archive.header.author,
        xor_enabled=xor,
        include_footer=footer,
        timestamp=archive.header.timestamp,
    )

    # 3. Write to disk
    with open(output_path, "wb") as f:
        f.write(output)

    # 4. Re-read and compare
    archive2 = parse_cfr(output_path)

    assert archive.header.num_records == archive2.header.num_records
    for r1, r2 in zip(archive.records, archive2.records):
        assert r1.name == r2.name, f"Different name: {r1.name!r} vs {r2.name!r}"
        assert r1.data == r2.data, f"Different data for {r1.name}"
        assert r1.rec_type == r2.rec_type, f"Different type for {r1.name}"

    print(f"Round-trip OK: {filepath} → {output_path}")
```

### Round-trip #2: Validation by the Original Binary

This is the most important test. Our Python parser could have a consistent bug (it writes incorrectly and reads incorrectly in the same way), so an internal round-trip is not sufficient. We need to confront our output with the reference binary:

```python
import subprocess

def test_binary_validation(cfr_path: str, binary: str = "./fileformat_O0"):
    """Verifies that a CFR archive is accepted by the original binary."""
    result = subprocess.run(
        [binary, "validate", cfr_path],
        capture_output=True, text=True
    )
    if result.returncode != 0:
        print(f"FAILURE: {binary} validate {cfr_path}")
        print(result.stdout)
        print(result.stderr)
        raise AssertionError("The original binary rejects our archive")
    print(f"Binary validation OK: {cfr_path}")

def test_full_roundtrip(filepath: str):
    """Full test: Python read → Python write → binary validation."""
    archive = parse_cfr(filepath)

    output_path = filepath + ".roundtrip.cfr"
    xor = bool(archive.header.flags & HeaderFlags.XOR_ENABLED)
    footer = archive.footer is not None
    output = serialize_cfr(
        archive.records,
        author=archive.header.author,
        xor_enabled=xor,
        include_footer=footer,
        timestamp=archive.header.timestamp,
    )
    with open(output_path, "wb") as f:
        f.write(output)

    # The original binary must accept our file
    test_binary_validation(output_path)

    # The original binary must also be able to read it and display the records
    result = subprocess.run(
        ["./fileformat_O0", "read", output_path],
        capture_output=True, text=True
    )
    print(result.stdout[:500])
    print(f"Full round-trip OK: {filepath}")
```

Let's run on our three archives:

```python
test_full_roundtrip("samples/demo.cfr")  
test_full_roundtrip("samples/packed_noxor.cfr")  
test_full_roundtrip("samples/packed_xor.cfr")  
```

### Round-trip #3: Generation from Scratch

The most ambitious test is to create an archive entirely from Python, without ever having read an existing file, and have the binary accept it:

```python
def test_generate_from_scratch():
    """Generates an archive from scratch and validates it with the binary."""
    records = [
        make_text_record("hello.txt", "Written entirely by our Python serializer."),
        make_binary_record("payload.bin", bytes(range(256))),
        make_meta_record("build.meta", {
            "generator": "cfr_parser.py",
            "chapter": "25",
        }),
    ]

    # Variant with XOR
    data_xor = serialize_cfr(records, author="pytest", xor_enabled=True)
    with open("/tmp/python_generated_xor.cfr", "wb") as f:
        f.write(data_xor)
    test_binary_validation("/tmp/python_generated_xor.cfr")

    # Variant without XOR
    data_plain = serialize_cfr(records, author="pytest", xor_enabled=False)
    with open("/tmp/python_generated_plain.cfr", "wb") as f:
        f.write(data_plain)
    test_binary_validation("/tmp/python_generated_plain.cfr")

    # Variant without footer
    data_noftr = serialize_cfr(
        records, author="pytest", xor_enabled=False, include_footer=False
    )
    with open("/tmp/python_generated_nofooter.cfr", "wb") as f:
        f.write(data_noftr)
    # Note: validate might warn about the missing footer,
    # but list/read should work
    result = subprocess.run(
        ["./fileformat_O0", "list", "/tmp/python_generated_nofooter.cfr"],
        capture_output=True, text=True
    )
    assert result.returncode == 0
    print(result.stdout)

    print("Generation from scratch OK (3 variants)")
```

If all three variants pass validation, our implementation conforms to the format.

## Debugging Round-Trip Failures

Round-trip failures are common during parser development. Here are the most frequent causes and diagnostic techniques.

### The Header CRC Does Not Match

Symptom: the binary displays `[FAIL] Header CRC`.

Likely cause: the range of bytes covered by the CRC is not what we think. Let's display the exact bytes:

```python
with open("output.cfr", "rb") as f:
    hdr = f.read(32)
print("CRC-ed bytes :", hdr[:16].hex())  
print("Stored CRC   :", struct.unpack_from("<I", hdr, 16)[0])  
print("Computed CRC  :", crc32(hdr[:16]))  
```

### A Record's CRC-16 Does Not Match

Symptom: the binary displays `[FAIL] Record N CRC-16`.

Likely cause #1: the XOR/CRC order is reversed. The CRC must be computed on the original data (before XOR), not on the stored data (after XOR).

Likely cause #2: the CRC covers the name + data, but we forgot the name or included an extra byte (for example the null terminator, which does not exist in this format — names are not null-terminated in the file).

Diagnostic technique — compare byte by byte what our serializer writes with what the binary writes for the same data:

```python
with open("samples/packed_noxor.cfr", "rb") as f:
    original = f.read()
with open("output.cfr", "rb") as f:
    ours = f.read()

for i, (a, b) in enumerate(zip(original, ours)):
    if a != b:
        print(f"Difference at offset {i:#06x}: original={a:#04x} ours={b:#04x}")
```

The first differing byte points directly to the problematic field.

### The `data_len_xor` Does Not Match

Symptom: the binary displays `[FAIL] Reserved check`.

Cause: we reversed the operation or forgot a record in the XOR. Let's verify:

```python
dlx = 0  
for rec in records:  
    dlx ^= len(rec.data)
    print(f"  {rec.name}: data_len={len(rec.data)}, cumulative dlx={dlx:#010x}")
print(f"final dlx = {dlx:#010x}")
```

### The Footer Total Size Does Not Match

Symptom: the binary displays `[FAIL] Footer total_size`.

Cause: the computation `len(payload) + FOOTER_SIZE` is off. Let's verify that `payload` indeed contains the header + all records (without the footer).

```
└── CLI (__main__)
    ├── parse <input.cfr>           — display the contents
    ├── validate <input.cfr>        — verify integrity
    ├── create <output.cfr> [files] — create an archive
    └── roundtrip <input.cfr>       — round-trip test
```

The CLI section makes the parser directly usable:

```bash
$ python3 cfr_parser.py parse samples/demo.cfr
$ python3 cfr_parser.py create /tmp/test.cfr file1.txt file2.bin
$ python3 cfr_parser.py roundtrip samples/demo.cfr
```

---

## What the Implementation Confirms

Writing the parser and serializer is the moment when every remaining ambiguity must be resolved. Questions that could remain vague in the `.hexpat` pattern ("the CRC probably covers these bytes") become binary choices in the code (the value is correct or it is not).

The round-trip validated by the original binary is the strongest proof of format understanding. It is a test that forgives no approximation: a single wrong byte and validation fails. If all three variants (round-trip, generation with XOR, generation without footer) pass, our model of the CFR format is complete and correct.

One last step remains: formalizing this understanding into a standalone specification document — that is the subject of section 25.5.
